## 二、對專案有用的新資訊（就兩點）

掃了 6 篇最技術性的檔案後，**真正影響架構的新資料只有這兩條**：

### 2.1 部首 4 大類分類系統（摘自 `部首分類0104.doc`）

這是一個我們先前資料沒有的**新分類維度**，可以加到 L2 字根的 metadata：

| 大類 | 子類 | 字根範例 |
|---|---|---|
| **本存**（自然存在，有機形） | 本體 | 日月金水土气玉石山 |
| | 植物 | 木禾黍麻米麥瓜竹艸 |
| | 肢體 | 目舌身頁肉血羽皮毛爪骨鼻角心手足髟耳尸齒 |
| | 動物 | 魚龜鼠鳥黽鹿隹虍貝豸豕羊虫犬馬牛 |
| **人造**（人造物，幾何形） | 食衣 | 臼缶豆皿酉衣巾革韋糸 |
| | 住行 | 門龠瓦車舟几广宀冂邑阜 |
| | 起居 | 鼎鼓鬲爻隶斤斗寸 |
| | 工作 | 耒聿网殳矢矛匕弓刀 |
| **規範**（抽象概念） | 定義 | 歹方齊高面首長工么囗士夊小大里疋 |
| | 根源 | 力光色音彡香火 |
| | 現象 | 谷黃雨青赤白夕鬼風穴疒黑巛 |
| **應用**（感應關係） | 訊息 | 言采老辛甘爿片玄冫曰文無龍 |
| | 事理 | 彳鬥非辵走行見用比支攴八飛止艮 |
| | 關係 | 生父子女氏干欠隶臣自至入立又 |

**對我們的意義**：每個字根都有自然/人造之別，這對**渲染策略選擇**有啟示：
- 本存字根（自然形）→ 傾向有機曲線、不規則粗細
- 人造字根（幾何形）→ 傾向規則直線、對稱結構
- 規範/應用類（抽象）→ 按既有筆順照畫即可

將來做「手寫風格化」或「字型美化」時這個分類會派上用場。目前 Phase 1-3 用不上，先存為 metadata。

### 2.2 漢字基因的 6 大功能（摘自 `倉頡輸入0022.doc`）

作者明確定義漢字基因有六項功能：

| 功能 | 語意 | 我們的專案是否需要 |
|---|---|---|
| **字碼** | 倉頡碼當索引 | 可選（作輔助 key） |
| **字序** | 筆順 | ✅ 核心 |
| **字形** | 向量形狀 | ✅ 核心 |
| **字音** | 讀音 | ❌ 本專案不處理 |
| **字辨** | 消歧義 | ❌ |
| **字義** | 語義 | ❌（5000.TXT 已附帶，可留著） |

**對我們的意義**：專案邊界明確為「字序 + 字形」兩項。之前架構規劃的範圍剛好。

## 四、架構結論（不變）

本批資料**不改變**前三份分析的結論 —— 路線 D 依然成立：

```
L3 會意字 (3,645 字)    ← 5000.TXT  (DB)
L2 象形/指事 (280 字)   ← 5000.TXT + 2018 部首分類  (atom + category)
L1 筆畫原子 (9+ 種)      ← MODVEC.ASM (parametric stroke)
L0 向量輸出              ← 新寫 (SVG / G-code / JSON)
```

唯一的小調整：**L2 層多加一個 `radical_category` 欄位**：

```python
@dataclass
class RadicalRoot:
    char: str
    category: Literal['本存', '人造', '規範', '應用']
    subcategory: str     # 例如 '本體' / '植物' / '食衣' / '定義' / ...
    strokes: list[Stroke]
    bbox: BBox
```

這個 metadata 在**第一階段先記錄，不影響渲染邏輯**；日後做風格化時才啟用。
