## 重要里程碑

### Multi-kind dispatch 通用化（r28）

```python
VALIDATORS = {KIND_PSD: parse_and_validate_psd,
              KIND_MANDALA: parse_and_validate_mandala}
SUMMARIZERS = {KIND_PSD: summarise_psd,
               KIND_MANDALA: summarise_mandala}
```

加新 kind 只改兩個 dict + 寫對應 parse/summarise function。`create_upload` 邏輯不動，這是 by-kind dispatch dict 取代 if/elif 的勝利。

### 並行 fetch 設計紅利（r29d→r29f→r29g）

`refresh()` 函式集中所有 fetch（me / list / profile / deeplink upload）進 `Promise.all`，後續 phase 直接 享免費 — r29f 加 hash deeplink 進入時 0 額外 round-trip，r29g 加 deep-link upload prepend 也只是 push 一個 fetch 進 array。

### Hash route (r29f) — 解三條痛

1. Reload 失憶 → 解
2. 無法分享 profile 連結 → 解
3. 未來 deep-link 基礎建設 → 鋪完底

`hash.mjs` 是 pure helpers，browser + Node 共測（`.mjs` 強制 ESM）。23 個 Node unit test 涵蓋 stateToHash / parseHash / round-trip / 防注入。後續 r29g 加 `upload` key 是純擴 1 行 whitelist。

### Reconnaissance 修正規劃（r29i）

開動 phase 5c 「profile 編輯」前先 grep 既有 implementation，發現 9/9 元件早已 shipped — 只缺 banner ✏️ 快捷入口。誠實 push back 重新 scope 比悶頭做好。

### Avatar 完整 plumbing（r29j+r29k）

- DB 存 nonce 不存 path（路徑可從 user_id 算）
- `_user_dict_with_avatar` 集中派生 `avatar_url` — 5 個 SELECT path 全走同 helper
- Cache-bust 用 versioned URL（`?v=<nonce>`）+ `Cache-Control: immutable` — 0 round-trip revalidate
- Initials fallback 8 色 palette + stable hash by user.id — 同 user 永遠同色
- Pillow verify + 重 open 是 image upload security pattern
- r29k 加 client validation mirror server 規則 + drag-drop on preview area

## 流程亮點

### 每 phase 標準節奏

每個 phase 都走同一條流程：

1. **規劃**：5-7 個架構 Q & A，每個標 ★ 推薦（reasoning 寫出來）
2. **確認**：user 「全 ★ OK」或挑 specific Q 改方向
3. **實作**：批次 edit，邊做邊 smoke test 早抓 regression
4. **驗證**：Node + Python 全跑一遍 / manual E2E checklist
5. **Bump version + decision log**：版本 + `docs/decisions/<date>_phase5b_<phase>_<topic>.md`
6. **Commit**：rebuild git index → batched add → conventional message
7. **User push**（sandbox 沒 SSH）

### Reconnaissance-first

開新 phase 前 grep 既有 code（r29i 教訓） — 避免重做輪子或誤判 scope。

### Pure helpers 抽 `.mjs`

4 個 module 都遵循「side-effect-free logic 抽 export」pattern：
- `hash.mjs` (r29f) — 23 tests
- `toast.mjs` (r29h) — 9 tests，含 `_toastSpec` pure helper
- `avatar.mjs` (r29j) — 8 tests，含 `_initialsSpec` pure helper
- `avatar.mjs` (r29k) — 9 tests，加 `validateAvatarFile`

DOM-coupled module 仍可 unit test — 把 input validation 抽 pure function，DOM 操作走 manual E2E。

## 教訓 / 工程習慣

跨 15 phase 累積的核心教訓彙整於 [docs/PRINCIPLES.md](../PRINCIPLES.md)（共通性原則文件）。

幾個直接可記的：

- **Reconnaissance 在 phase plan 之前** — 開新 phase 前 grep 既有 implementation，避免重做輪子或誤判 scope
- **多入口共用 single execution path** — file input + drag-drop 兩入口都走 `_handleSelectedFile`
- **Pure logic 抽 `.mjs` export 給 Node test** — DOM-coupled module 中的 logic 抽出來，跨 4 個 module 同 pattern
- **Client mirror server validation** — single source of truth，先寫 server 再 mirror（含 charset 邊界 case）
- **Cross-cutting derivation 集中 helper** — `_user_dict_with_avatar` 5 個 SELECT path 全走同一函式，避免 multi-point drift
- **DB column 應存 unique fact 不存 derivable** — `avatar_path` 存 nonce 不存路徑（路徑可從 user_id 算）
- **Versioned URL > ETag 對 cache-bust** — `?v=<nonce>` + `immutable` = 0 revalidate round-trip
- **Whitelist > blacklist 防注入** — `parseHash` 對 sort/kind enum 用 includes，user int 用 `Number.isInteger && > 0`
- **Empty state 是 affordance hint** — 空 bio placeholder / initials fallback，比 hide 友善
- **`requestAnimationFrame` 包 post-mutation scroll** — `innerHTML = ...` 後 layout 未 settle，等下一 frame 才 measure / scroll
- **CSS 多重 cue 對重要狀態** — outline + scale + label 比 single-style 訊號強
