## 二、三個主要工作階段

### 階段 1：抄經模式精修（5bv → 5cc）

針對既有抄經模式做針對性視覺優化，**修了一個重大隱形 bug**：隸書/篆書字形在抄經頁完全空白渲染。

| Phase | 工作內容 |
|---|---|
| 5bv | `render_sutra_page` 加 `mark_renderer` 參數，SVG 用 `<text>` 元素，PDF 強制 polyline（避免伺服器需要 CJK 字型） |
| 5bw | **修隸/篆 skeleton fallback**——`_char_cut_paths` 只看 `outline`，但 lishu/seal source 預設 skeleton 模式產出 `outline=[]` → 整頁空白。加 polyline fallback |
| 5bx | **二階 bug**：5bw 的 polyline 雖然 emit 了，但被 inner scale transform 把 stroke-width 縮成 0.0025mm（不可見）。加 `vector-effect="non-scaling-stroke"` |
| 5by | skeleton stroke-width 從固定 0.55mm 改成 `char_size × 0.12` 比例 |
| 5bz | 加 `outline_glyph_loader` 跟 reference 字形層；SVG 下載保持純骨架，預覽和 PDF 顯示參考字形 |
| 5ca | 雙層 opacity 統一 0.55（列印描紅一致灰度）|
| 5cb | 後續調整：skeleton 0.55 → 0.03（隱形）+ reference 0.55（主體），讓 reference 主導視覺 |
| 5cc | 隸/篆 字型選擇時 SVG 警語（骨架輸出、罕用字未收錄）|

**關鍵教訓**：5bw → 5bx 暴露我「測試只證明形式正確，沒證明效果正確」的問題。後來規則改成：任何 SVG `transform` + `stroke` 修改，必須做兩層驗證——DOM 層（屬性正確）+ 像素層（cairosvg raster 採樣）。

### 階段 2：筆順練習頁（5d）— v0.12.0

獨立 SPA `/handwriting`，使用者可在 Canvas 上書寫，軌跡含 **時間戳 + 筆壓 + 觸控筆角度**，本機 IndexedDB 儲存，能匯出可直接餵寫字機器人的 Plotter SVG。

完整 14 條 task：
- 後端：`GET /handwriting`、`/api/handwriting/reference/{char}`、`/api/sutra/text/{preset}`
- 前端 6 個 ES module：`canvas.js`（PointerEvents + DPR + 雙層）、`storage.js`（IndexedDB v1）、`materials.js`（4 種素材源）、`grid.js`（7 種格線）、`reference.js`（字形 fetch + 渲染）、`exporter.js`（JSON / Plotter SVG / 自寫 ZIP encoder）
- 修了 5d-7-bugfix（reference 字形 bbox-center 對齊 + canvas resize 事件）

**核心設計決策**：
- ZIP encoder **自寫不依賴 JSZip**（120 行 vs 90KB vendor），通過 Python `zipfile` round-trip 驗證
- 軌跡座標 normalize 到 EM 2048（與內建 stroke-order 引擎相同座標系）
- 筆壓 fallback `0.5`（滑鼠/手指沒實際 pressure）
- iOS Safari 觸控防呆：`touch-action:none` + `overscroll-behavior:none` + LINE/FB/IG 內建瀏覽器警告

### 階段 3：公眾分享庫（5g）— v0.13.0

獨立 SPA `/gallery`，使用者可上傳 PSD JSON 分享給公眾，**email magic-link 登入**，SQLite 後端。

完整 13 條 task：
- 後端 5 個檔案：`config.py`、`db.py`、`smtp.py`、`auth.py`、`service.py`
- 11 個 API endpoints（auth 3 + me 2 + uploads 5 + page 1）
- 前端：`gallery.html` + 4 個 JS module（`gallery.js`、`auth.js`、`uploader.js` + 共用）
- 部署文件 `docs/GALLERY_DEPLOYMENT.md`

**核心設計決策**：
- **零第三方 auth 依賴**：HMAC-SHA256 自簽 token（無 PyJWT）、stdlib smtplib（無 aiosmtplib）、SQLite（無 ORM）
- **防偽造機制**：magic link 即使 HMAC 對得起來，沒在 `login_tokens` 表登記就被拒——攻擊者拿到 secret 也無法獨自簽出可用 token
- **一次性 magic link**：consumed 旗標防 email 前轉重放
- **Dev mode**：`AUTH_DEV_MODE=true` 不需 SMTP 即可全流程測試
- **session 用 SQLite 不是 JWT**：撤銷只要一句 `DELETE`
- **TestClient cookie jar isolation**：多使用者測試用 per-user `TestClient` 而非 per-request `cookies=` kwarg

## 三、累積到的設計原則 / 經驗

這些是這幾天碰到 bug + 修法歸納出來的、可以套用到未來的規則：

### SVG 渲染相關

1. **任何 `<g transform="scale()">` 內的 `stroke`**：要嘛用 `vector-effect="non-scaling-stroke"`，要嘛把 stroke-width 預先乘以 inverse scale。否則細線會被縮到亞像素消失。
2. **任何 outline 渲染**：先檢查座標系是 EM 全填還是有 padding/skew。多種字型源（kaishu / lishu / seal_script）的 EM bbox 中心可能不在 (1024, 1024) → 要 bbox-center 對齊，不能直接 EM (0..2048) → canvas (0..w, 0..h)。
3. **抄經頁面的 reference 字形 + skeleton 雙層**：適合「列印描紅」用途的灰度約 12%（`#cccccc × 0.55 ≈ #e1`），符合書法描紅紙觀感。

### 測試 / 驗證相關

4. **測試強度層次**：「DOM 屬性正確」≠「視覺看得見」。SVG transform + stroke 類修改，必須加 raster sanity 採樣（cairosvg → PNG → Pillow getpixel）。
5. **TestClient cookie jar 是 per-client 持久的**，不是 per-request。多使用者場景**用 per-user TestClient**，不要在單一 client 上交替切換 `cookies=` kwarg（已 deprecated 且行為不可預期）。
6. **path 參數的 default 值在 FastAPI**：`Path(...)` 不能有 default，但 `Path` 與 `pathlib.Path` 命名衝突，必須 `from fastapi import Path as ApiPath`。

### 架構 / 取捨相關

7. **vendor 檔案 vs 自寫的取捨**：JSZip 90KB vs 自寫 120 行 STORE-mode encoder——對「只需要產生不需要解析」的場景，自寫贏在零依賴 + 部署乾淨。
8. **dev mode 的價值**：5g 的 SMTP 在 sandbox/CI/初次部署都缺。`AUTH_DEV_MODE=true` 印 console 取代寄信，**讓 5g MVP 能在不設 SMTP 的環境就跑得起來**——大幅降低試用門檻。
9. **獨立 SPA 子頁路線**：5d (`/handwriting`) 跟 5g (`/gallery`) 都用「獨立子頁 + 自包含 ES modules + 純 fetch /api/* 後端」，不和主頁共享狀態。維護單純，使用者也可獨立書籤。

### 流程 / 規劃相關

10. **大功能先寫詳細 spec、確認後才動代碼**：5d 跟 5g 都是先列 P0 問題、給用戶確認、再開工。避免做到一半發現方向錯。
11. **Batch 切分按「使用者能感受到的里程碑」**：5d 切成 5 個 batch（基礎、書寫、素材模式、資料進出、警語+測試），每批結束都有可看到的進展。
12. **訊息文案的精確性**：使用者抓到「您寫的每一筆都會幫助累積真實軌跡資料庫」的不準確（其實只存本機）→ 改成明確說「先存本機資料庫，日後可選擇匯出/提交」。
