**r2（4 項 patch）— commit 55ebd13 / v0.14.20**

User 反饋：「上弧 / 下弧文預設改範例文字 / 調整橢圓比例 / 修正字間距視覺差」。

最重要的：**字間距視覺差** — user 觀察「中央的字間距比邊緣寬」。這是因為原本 angular-equal 分布在橢圓上，arc length per radian = √(a²sin²t + b²cos²t)，t=±90° (apex) 時 = max(a,b)，t=0/π (sides) 時 = min(a,b)。a > b 時 apex 處每角度步進對應更長弧長 → 視覺寬。

**解法：弧長均分（arc-length parameterization）**
- 新增 `_ellipse_arc_length_table(t_start, t_end, a, b, samples=200)` 用梯形法 numerical integrate
- 新增 `_t_at_arc_length(target_len, ts, cumlens)` 反查 t
- `_oval_arc_positions` 改用：每字 i ∈ [0, n-1] 算 target = (i / (n-1)) × total_arc，反查得 t_i

驗證：11 字均分後 pairwise Euclidean distances max/min ratio = 1.007（接近完美等距，was much higher with angular-equal）。

橢圓比例改 50×30 (1.67:1) → 50×35 (1.43:1)，更貼近業界 4-6cm 級橢圓章。

145 tests（+ 2 new uniform-spacing tests）。

## 二、本日學到的規則

### 結構化 input vs separator hack

當一個功能 input 有多個語意獨立欄位（如橢圓章 5 段：上弧 / 3 行 body / 下弧），用結構化 schema（`oval_arc_top`、`oval_body_lines: list[str]` 等）優於 single text + separator hack（如 `text="..|..|.."`）。

理由：

- **User mental model**：UI 5 個獨立 input 直觀，user 不需學 separator 規則
- **語意清楚**：API field 名稱自我文件化
- **可選欄位處理**：empty string 自動 skip，不需 escape
- **未來擴充**：加新欄位（如 `oval_arc_top_size_mm`）容易，不需 break 現有 separator schema

代價：API surface area 略大（多 3-5 個 fields），但 backward compat 容易維持（all empty → fallback 既有 single-text 邏輯）。

→ candidate **§8.13**。

### UI 預設要 hook init time，不只 change event

當 UI 控件的 default state 依賴 JS 在 preset/select 切換時填入時，**必須在 init 時呼叫一次**。光是 hook `change` event 不夠 — 第一次載入或 user 沒做 change 時不會觸發。

具體模式：
```js
$("st-preset").addEventListener("change", stampApplyPresetDefaults);
stampApplyPresetDefaults();   // ← 缺這行就會有 init-time inconsistency
```

症狀：user 截圖顯示某個依賴 default 的 UI 元素（checkbox / hint / size）跟 expected 不一致。Likely 不是 deploy 問題，是 init time hook 漏掉。

→ candidate **§8.14**。

### 弧長均分（arc-length parameterization）

橢圓 / 不對稱曲線上分布元素時，等角度步進 ≠ 等視覺距離。`ds/dt = √(a²sin²t + b²cos²t)`，apex（短軸方向）跟 sides（長軸方向）有 a/b 倍差距。

修法：numerical integrate 建 (t, cumlen) lookup table，反查每字 target 弧長對應的 t。實測 11 字 max/min ratio 從原 angular 的不均勻 → arc-length 後 1.007（基本完美）。

這個 pattern 在 SVG / canvas 動畫、地圖路徑取點、字體 hinting 都通用。**不寫 playbook**（太 niche），但記在這次決策紀錄裡。

### Cross-session git index.lock

在 cowork session 動主分支時，git index.lock 可能因 session 隔離而被 stale。`git status` 還能跑（只讀），但 commit 失敗。User 端 terminal 不受影響，所以**檔案改動仍在硬碟上**（mounted folder），只要 user 自己 commit/push 就部署得到。

→ 跟既有 `feedback_cross_session_race.md` memory 連動，加備註。
